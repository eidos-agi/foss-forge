## North Star

foss-forge exists for one reason: **make sure the agentic tools you build get discovered, trusted, and adopted.**

Building great software is necessary but not sufficient. The graveyard of open source is full of excellent tools nobody heard of. foss-forge ensures that everything Eidos AGI ships is properly marketed, professionally presented, and credibly positioned — so the work speaks for itself and the world actually sees it.

## What foss-forge Actually Cares About

In priority order:

### 1. Marketing & Discoverability (the point)
- README that converts visitors to stars in 5 seconds
- Badges, star history charts, demo content above the fold
- Launch playbook: HN, Reddit, Product Hunt, awesome lists
- Social cards and Open Graph images for link previews
- SEO: description, keywords, comparison tables vs. alternatives
- Distribution: PyPI, Homebrew, pipx, uvx — be where people look

### 2. Trust & Credibility (prerequisite)
- LICENSE (you can't adopt what you can't legally use)
- CHANGELOG (shows the project is alive)
- CONTRIBUTING + CODE_OF_CONDUCT (signals a real community)
- SECURITY.md (shows you take it seriously)
- CI badges (proof it works)

### 3. Agentic Quality (differentiator)
- Tool descriptions that help agents choose the right tool
- Parameter schemas that agents can parse without guessing
- Error messages that tell agents what to do next
- MCP config examples in README

### 4. Engineering Basics (table stakes)
- pyproject.toml with full metadata
- CI/CD workflows
- Publish pipeline
- Dependency hygiene

## What foss-forge IS

- Skills that an agent runs to audit, scaffold, release, and launch FOSS projects
- Templates for community health files and CI/CD
- A marketing playbook encoded as agent knowledge
- A forge-factory (`/forge-init`) for creating new forges

## What foss-forge is NOT

- Not primarily a code quality tool — that's what linters and test-forge are for
- Not a CI system — it generates CI configs but doesn't run them
- Not demo-forge — it delegates visual content creation there

## Success Metric

Every Eidos AGI public repo has a README that makes you want to star it, a launch strategy that puts it in front of the right people, and trust signals that make adoption a no-brainer.
